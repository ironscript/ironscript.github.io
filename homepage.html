Ironscript is a minimal LISP like programming language implemented in Javascript. It is interpreted and the interpreter is written in ES2015. Currently we have an interpreter that runs on browser and nodejs environments and a bundler to bundle ironscript modules into .js files for the browser.

Ironscript IS

LISP and Javascript Mixed.
Ironscript is essentially an S-Expression evaluator written in Javascript. This evaluator can use functions written Javascript and interact with the JS environment at runtime through those functions. In fact you can write Javascript functions inside your Ironscript S-Expressions using a simple API.
Functional.
Ironscript is essentially a LISP over JS, every valid Ironscript statement / expression / program is an S-Expression. You will be writing mostly functions and streams to express your logic in Ironscript.
Better suited for expressing Asynchronous data flows.
The S-Expression evaluator is written in CPS (Continuation Passing Style). Therefore it's a tiny bit slower than synchronous Javascript. But it enables the programmer to express complex asynchronous data and control flows linearly.
Modular, Easy to Learn, Use and Extend.
Ironscript provides an easy-syntax module in the standard library (stdlib), that provides some frequently used syntactial abstractions as pattern rewriting rules. Ironscript also provides an easy construct for defining pattern rewrite rules.
Free and OpenSource
Free as in 'freedom'. Ironscript is licensed MIT and the source is hosted on github.

Ironscript IS NOT

CommonLISP or Scheme or ...
Ironscript is a LISP with its own syntactical conventions. Ironscript is not designed to be pure LISP or Pure Javascript. It's designed to be minimal, easily extendable and suitable for Rapid Prototyping of applications. Its design goals are not syntactical or theoretical purity. We are more concerned about simplicity and extensibility.
Compiled to Javascript. (But will be)
Currently we do not have an Ironscript to JS transpiler. We do hope to have a decent Optimizing Compiler to JS from v1.3 onwards. Currently there is a lot of room for optimizations to be exploited, the interpretation is reasonably fast but with an Optimizing Compiler we can generate better Javascript.
Limited to the Browserland
Ironscript can run 'literally' anywhere, where Javascript can run. That includes servers via NodeJS, mobile applications via Cordova / PhoneGap. And as for the browser, this website is written in Ironscript. Ironscript has been tested on Cordova too.
Stable Yet.
As of March 2017 (i.e. when this webpage was written), all these 'we' in this document is a single person. Therefore the pace of development is slow and the design goals are driven by the needs of an individual. Contribute to make it better and more stable.

Features

Functional

LISPlike functional with higher order functions.

Modular

Organize your code into modules and reuse.

Asynchronous

Built around a CPS S-Expression evaluator.

Streams

Streams are live values or values that change asynchronously and are first class citizens in Ironscript.

Dynamic

Ironscript is Dynamically Typed and almost eveything that's not literal is symbolic.

Rapid App Dev

Prototype highly interactive apps faster and more easily.

Extendable

Ironscript is easily extendable through Macros, Rewrite Rules and Javascript.

Minimal

The Ironscript interpreter and browserside runtime are about 12KB combined.

Mix with JS

Interoperable with Javascript through a powerful embedding syntax and API.

Get Started
Installation with NPM
Ironscript is can be installed using npm. The package adds 2 commands to the terminal, ironscript and 
ibun. It's recommended to install the package globally by using the -g flag. You may have to 
use sudo for installation on Linux depending on your permissions.
npm install -g ironscript
Doing the Hello World !
First check the Hello World example in the Try It Online section, create a file named hello.is on your system, write the code and
run ironscript hello.is from your terminal. It's that easy, really. We do not have an REPL yet, we hope to have 
implemented an REPL by version 1.3 .
